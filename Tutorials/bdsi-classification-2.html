Review¶

  • Assume there is an underlying function $f$ relating predictor variables (features) to the response (class).
  • $f$ unknown, so need to estimate it from data.
    • parametric methods
      • assume the shape of $f$, so that estimating of $f$ is equivalent to estimating a set of parameters
      • e.g. LDA (normality, equal feature variances b/w classes)
      • e.g. Logistic regression (log-odds ratio of $P(Y=j|X=\widetilde{x})$ is a linear function of $\widetilde{x}$)
      • e.g. QDA
    • nonparametric methods
      • e.g. kNN
      • e.g. Random Forests

Review (cont'd)¶

  • The optimal classifier computes $P(Y=j|X=\widetilde{x})$ for all classes $j$ and classifies a test observation to the class giving the largest conditional probability.

  • Many classifiers try to estimate $P(Y=j|X=\widetilde{x})$ to classify a test observation

    • kNN uses (training set) neighbors of the the test observation to estimate $P(Y=j|X=\widetilde{x})$
    • Logistic regression tries to model the log odds of $P(Y=j|X=\widetilde{x})$ as a function of $X$
    • LDA, QDA model the $P(X| Y=j) \sim N(\mu_j ,\sigma^2)$ and uses Bayes' rule to flip this probability into $P(Y=j|X=\widetilde{x})$

  • Linear classifiers define linear decision boundaries, nonlinear classifiers define nonlinear decision boundaries

Revisiting Random Forests¶

Decision trees¶

  • A decision tree comes up with a series of rules to "recursively divide the training data into buckets of homogeneous members through the most discriminative dividing criteria possible."

    • When the response is categorical, the decision tree is called a classification tree
    • These rules are nodes in the tree
    • Every rule consists of a variable and a "cut point" (e.g. maxBP<120 for the heart disease dataset), leading to binary splits
    • The buckets we end up with are called leaves

  • Once we're done forming buckets, we make the same prediction for every observation in a given bucket

  • Therefore, having buckets that are homogeneous or more "pure" will lead to lower test error rate

How to grow a decision tree?¶

  • At every step, we select the rule that maximizes gain in "purity" (as measured by Gini index or entropy)
  • This is a greedy approach
  • Stop growing when splitting the tree further does not result in significant increase in purity

  • What do the numbers at each node mean?
    • At the top node, we see "125/26" and 0. This means that at this node we have 125 observations from class 0 and 26 observations from class 1. The "0" means that if this bucket were a terminal node, we'd classify all observtions in this bucket would be classified to "0".

Advantages of decision trees¶

  • Interpretable (easy for humans to understand)
  • Can handle categorical and continuous predictors without need for dummy variables
    • e.g. in linear regression, we can't directly use eyecolor = ("black", "brown", "brown", $\ldots$ , "green") as a predictor, but with trees we can

Disadvantages of decision trees¶

  • Not very robust
  • If you want to incorporate new data, you have to train the decision tree all over again
  • Generally very poor classification performance

Random Forests¶

  • Is a "strength in numbers" approach to using decision trees (ensemble method)

  • Takes random subset of the training data and a random subset of features and uses it to grow a decision tree.

  • Repeats this many times, resulting in many trees

  • "Combines" the trees

To ponder...¶

We saw that nonlinear classifiers produce more flexible classification boundaries. Why might we still want to use a linear classifier over a nonlinear classifier?

  • Nonlinear classifiers may overfit the training data and fail to generalize well, leading to poor predictive accuracy on the test data. Linear classifiers define linear decision boundaries (hyperplanes that separate observations into one class or the other), and may resist overfitting.
  • Another way to understand overfitting: low bias at the expense of high variance

  • If your classifier has a low training error rate but high test error rate, overfitting may be to blame

To ponder...¶

Today we didn't discuss the actual datasets you'll be working with. It turns out that some of these classifiers don't work well when applied to those datasets.

  • Which ones?
  • Can you think of some reasons why?

In your datasets, the number of features ($p$) is much larger than the number of training observations ($n$). We say this data is high dimensional.

Many traditional statistical methods, including almost all the classifiers we covered, "do not work" on high dimensional data.

Why not?¶

With high dimensional data:

  1. We tend to overfit classifiers
  2. Parameter estimation may be an ill-posed problem

Overfitting¶

  • When the number of features is large relative to the number of training observations, our (training) observations are basically a few points sparsely populating some very large feature space
  • Since there's more space between the data points, it's easy to find a surface that perfectly separates them
  • But this surface is actually really complicated "when we look at it in 2D" - overfitting!
  • This is the curse of dimensionality

Ill-posed parameter estimation problems¶

  • the number of parameters we need to estimate outnumbers the number of training observations
  • e.g. LDA requires us to estimate $\pi_0, \pi_1, \Sigma, {\mu_1}, \mu_2$. The covariance matrix $\Sigma$ alone is $p \times p$. If we have less than $p$ data points, then the problem of estimating $\Sigma$ is ill-posed!

  • Suppose we only have one predictor. LDA assumes that an observation $X \in \mathbb{R}$ coming from class $j$ is distributed $N(\mu_j, \sigma^2)$.

  • If we have $p$ predictors, LDA assumes that an observation $X \in \mathbb{R}^p$ coming from class $j$ is distributed according to $\mathcal{N}_p(\mu_j, \Sigma)$ (multivariate normal)

  • This is the analogue of the $N(\mu, \sigma^2)$ density in $p$-dimensions: $$f(x) = \frac{1}{(2\pi)^{p/2}~|\Sigma|^{1/2}} \exp \left( -\frac 12 (x-\mu)^\top \Sigma^{-1} (x-\mu) \right)$$

    • $\mu_j = E(X)$ is the mean of $X$, a $p$-dimensional vector
    • $\Sigma = cov(X)$ is the $p \times p$ covariance matrix of $X$

  • The estimates for $\mu, \Sigma$ are analogues of our familiar one-dimensional estimates.

  • $\Sigma$ is $p \times p$, so this already gives us $p^2$ parameters we need to estimate.
  • Since the number of training observations is less than $p$, we have more parameters to estimate than we do data points! This is what we mean when we say a parameter estimation problem (here, for $\Sigma$) is ill-posed.

What can we do?¶

We can alleviate these problems with

  1. Feature selection: Judiciously select a subset of features to use so that the number of features is less than the number of training observations.
  1. Regularized methods: Regularized methods use additional information or make additional assumptions in order to reduce overfitting and/or overcome ill-posed parameter estimation problems
    • e.g. impose penalties on objective function when doing parameter estimation
    • Penalties are controlled by regularization parameters, which are generally selected via cross-validation

Feature selection for classification¶

  • Some classification methods do this automatically, as part of the method ("embedded methods")
    • e.g. Random Forests
    • $\times$ May not be as transparent, since it is happening "inside" the classifier
    • $\checkmark$ Easy for human user
  • Sometimes you do this before applying a classifier ("filter methods")
    • e.g. select genes which are most "different" between the two classes, then apply any classifier you wish
    • We will cover this by example when we walkthrough a classification analysis.
    • $\times$ Does not consider correlations between variables, so may select redundant variables (often OK - why?)
    • $\checkmark$ Tend to be robust, computationally efficient
  • Is performed using the training set only!</span>

Regularized methods¶

We'll cover:

  1. Diagonal LDA
  2. Regularized LDA
  3. glmnet

The first two are regularized versions of LDA. The last is a regularized version of logistic regression.

Diagonal LDA (DLDA)¶

  • Assume that features are independent, conditional on class
    • e.g. Given that someone has heart disease, their resting blood pressure is independent of their cholesterol level, max heart rate, etc.
    • This is equivalent to assuming that $\Sigma$ is diagonal (why?)</span>
  • This assumption corrects ill-posedness by greatly reducing the number of parameters we have to estimate (why?)
    • cov(X) will be a better estimate of $\Sigma$ (it will now be invertible)
    • But cov(X) may still not be a good estimate of $\Sigma$
      • Solution: DLDA with "top 10 genes" (more on what "top 10 means later")

Regularized LDA¶

  • "Nudge" the covariance matrix
    • Use `cov(X)` + $\alpha I$ instead of `cov(X)` as our estimate of $\Sigma$
  • "Nudging" corrects ill-posedness of estimating $\Sigma$, since `cov(X)`+ $\alpha I$ is always invertible.

Regularized logistic regression (glmnet)¶

  • Incorporate "penalities" into parameter estimation process
  • These penalties make parameter estimates slightly biased, but greatly reduce their variance, so overall, parameter estimates are improved

How to perform a classification analysis¶

  1. Preprocessing (e.g. cleaning, transform, handling missing values)
  2. Feature selection (optional)
  3. Train classifier
  4. Evaluate/test classifier

Train a classifier on a training set , evaluate it on a test set .

We have the class label of every observation in the training and every observation in the test set.

The practical value of building a good classifier is that we can deploy it and use it in real life, on observations we have no class labels for. A classifier with low error rate on the test set (observations we're pretending we've never seen before) hopefully also has low error rate on observations it truly has never seen before.

Feature selection: one method¶

  • We might choose to select the 10 genes which are most differentially expressed between people for whom the drug was not effective ("nonresponders") and people for whom the drug was effective ("responders").
  • "10 most differentially expressed genes" $\leftrightsquigarrow$ genes with 10 lowest p-values
  • Use the training data to do compute p-values. Once you've identified the "top" genes, make sure to subset both training and test sets to these genes.
  • Reminder: The only time you should touch the test data is at the very end, when you evaluate your classifier

Evaluating a classifier¶

  • Using the previously trained classifier, make predictions for the test set.

    • In this step we take the $\widehat{f}$ estimated from the training set and applying it to the test set observations $\{ \widetilde{x}_1, \widetilde{x}_2, \ldots, \widetilde{x}_{\widetilde{n}}\}$
    • Note that we are not using the true class labels of the test observations, $\{ \widetilde{y}_1, \widetilde{y}_2, \ldots, \widetilde{y}_{\widetilde{n}}\}$

  • Compare the predictions the classifier made for the test set observations to the true class labels of the test set observations and compute test error rate (or accuracy rate).

    • i.e. compare $\{ \widehat{f}(\widetilde{x}_1), \widehat{f}(\widetilde{x}_2), \ldots, \widehat{f}(\widetilde{x}_{\widetilde{n}})\}$ to $\{ \widetilde{y}_1, \widetilde{y}_2, \ldots, \widetilde{y}_{\widetilde{n}}\}$

How to choose a classifier¶

  • For hypothesis testing, we have many kinds of hypothesis tests...each suited for a particular situation.
    • e.g. paired t-test
  • For classification, we have many kinds of classification methods...but it's not easy to say which classifiers will be good in any given situation
    • We don't know if parametric assumptions on $f$ hold (e.g. $f$ linear?)
    • Even if an assumption doesn't hold, we may still achieve good prediction performance by using it
      • e.g. DLDA often works well, even though the independence assumption is probably not true!
  • In practice, we generally choose a classifier by training several different kinds of classifiers and picking the one that gives the smallest test error rate.
