Introduction to PCA¶

High-Dimensional Data¶

  • All of our genetic data (expression, methylation, copy number) are high-dimensional. This means the number of features (genes, DNA segments) is much larger than the number of cases (cell lines).
    • Expression: 850 x 17,737
    • Methylation: 850 x 485,512
    • Copy number: 850 x 46,221
  • We use the notation $n$ to represent the number of cases and $p$ to represent the number of features.
    • High-dimensional data is when $p > n$ (and often, $p >> n$).
  • In high-dimensional settings, PCA can be used as a method of dimension reduction -- identifying a lower-dimensional set of features from a very high-dimensional set.

Dimension Reduction¶

There are many methods of dimension reduction that are used with high-dimensional data.

  • Choosing a subset of the features that captures the variation in the data.
    • For example, suppose all cell lines have basically the same expression levels for 17,700 genes and very different expression levels for 37 genes. Then we can reduce the dimension of our data by only focusing on the 37 important genes.
  • Take linear combinations of the original features to create new features that capture the variation in the data.
    • The number of linear combinations may be much smaller than the number of original features.
    • This is a transformation of the data, which loses interpretability.

PCA in Two-Dimensions¶

Although we are talking about PCA as a dimension reduction technique, we will consider the two-dimensional case ($p = 2$) to begin with.

Example (from Introduction to Statistical Learning):

  • The data we will consider consists of population size and spending on advertisments for 100 small cities.
  • Plotted data appears in thousands of people for population size and thousands of dollars for advertising spending.

What is PCA?¶

PCA is a method that identifies the directions that contain most of the variation in the data.

  • first PC direction: the direction in which the data vary the most.
    • If we were to project the data onto that line, the projected observations would have greater variance than if we had projected them onto any other line.
  • second PC direction: the direction in which the data vary the most that is perpendicular to the first PC direction.

The maximum number of principal components is equal to the number of original features.

Mathematical Representation¶

The first principal component direction can be writted mathematically:

$$ Z_1 = 0.839 \times (pop - \overline{pop}) + 0.544 \times (ad - \overline{ad}) $$

  • This is the equation for the line in the first principal component direction.
  • The vector that results from plugging the data into this equation contains what are called the first PC scores. These are the projection of the data onto the first PC direction.

There is also an equation for the second principal component direction and a similar vector of second PC scores, but we will focus on PC1 because we want to reduce the dimension of our problem.

PC Scores¶

These are a transformation (a linear combination) of the original data.

  • Each of the elements in the vector of PC scores is a single number that summarizes the population size and the advertising spending for that city.
  • This is the idea of dimension reduction -- we have gone from two features describing our data (population size and ad spending) to one.
    • Ideally, that one new feature contains most of the important information from the two original features.

PCA in Higher-Dimensions¶

In higher dimensions, the idea is exactly the same, but it becomes much more difficult to visualize.

  • Each PC points in the direction of largest variation that is perpendicular to the previous PC directions.
  • The maximum number of PCs is the total number of features in the data ($p$).
  • A good number of PCs to use is a tuning parameter ($k$).
    • When choosing $k$, we want to balance simplicity and information loss.
    • How to choose $k$? Cross-validation!

Plotting Principal Components¶

Visualizing High-Dimensional Data¶

One of the first things to do when exploring data is to visualize it in as many ways as possible, but it can be difficult to know how to do that when the data is high-dimensional.

  • How can you plot 17,737-dimensional data?
  • What does 17,737-dimensional space really look like?

Plot PC Scores¶

Instead of plotting the original data, we can plot the PC scores of the first few principal components.

  • We can view an approximation of 17,737-dimensional data in two-dimensions.
  • This allows us to visualize variation in the data because the largest amount of variation is captured in the first few PCs.
  • We can look for clustering or separation in the data.
In [5]:
library(ggplot2)
options(repr.plot.width = 5, repr.plot.height = 3)
In [6]:

ggplot(data = pc.scores) + geom_point(aes(x = V2, y = V3, color = EFFECT)) + labs(x = "PC2", y = "PC3")

Conclusions¶

These plots, especially PC1 vs. PC2, indicate that a large source of the variation in the expression data is related to the effectiveness of drug 1014.

  • In the two directions of largest variation, we see separation between the effective and ineffective groups.
  • This tells us that the greatest variation in the expression data for these cell lines aligns with differences in drug effectiveness.
  • This indicates that the expression data may help us to classify this drug as effective or ineffective for various cell lines.

Principal Components Regression (PCR)¶

The Method¶

  • Find the first $k$ principal components of the features.
  • Use those principal components as predictors and regress the response onto them.

The Idea¶

  • We are assuming that the first $k$ principal components contain the relevant information about the response.
  • In other words, the directions of highest variability in the expression data, which are captured in the first $k$ principal components, are the directions associated with drug effectiveness.
    • If we have chosen a good predictor, this assumption turns out to be reasonably true.
  • If this assumption is true, then the results of the regression will be better than when using all the original predictors in the model.
    • $n > p$: we reduce the risk of overfitting by only estimating $k < p$ coefficients.
    • $n < p$: we cannot estimate more coefficients than the number of observations we have.

Conclusions¶

  • PCR can be a good approach if a small number of features contain the majority of the important information.
  • PCR doesn't do any variable selection.
  • PCR uses transformations of the features, so it lacks interpretability.
