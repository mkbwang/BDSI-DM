Bias-Variance and Over-fitting¶

Classification setting

  • We have a response vector $Y$ and a predictor matrix $X$ such that $Y = f(X) + \varepsilon$.
  • The function $f$ is unknown, so we estimate it with the data and then predict $Y$ from $X$: $\hat{Y} = \hat{f}(X)$.
  • Bias = how much error is introduced when approximating real-world data with a model
  • Variance = how much our estimate $\hat{f}$ would change if estimated from different training data.
    • Low variance means the estimate wouldn't change too much between different training sets -- this is what we want.
  • More flexible models have lower bias and higher variance.
    • When we overfit, we are using a model that is too flexible.
  • Less flexible models have higher bias and lower variance.

Introduction to Cross-Validation¶

Classification Goal¶

  • When we do classification, we are trying to build a model that will predict the class label (e.g. effective or ineffective drug) based on observed feature data (e.g. gene expression level).
  • We are interested in creating the best possible predictive model. But how do we characterize "best"? And how do we find the "best" model?
    • One way we do this is by looking at the error rate -- what fraction of observations do we misclassify?

Estimating Classification Error¶

If we are given a dataset where the class labels are known, we can build a classifier.

  • These data are used to fit the model and estimate parameters.
  • We can easily get the error rate of the classifier on this set of data:
    1. Use the classifier to estimate the class labels.
    2. Compare the estimated labels to the known labels.
    3. Calculate an error rate (# of observations misclassified/total # of observations)
  • However, since the classifier was trained on these data, this error rate is not a good indication of how well the classifier will fit new data.
    • The classifier usually works much better on the training data than on new data, giving us false confidence in the classifier's ability.
    • We underestimate the true error rate.

How do we fix this?

  • To get a good estimate of the error rate when the classifier is applied to new observations, we would like a set of data that is completely separate from the data used to train the model.
  • Then we can say that a model is good if it gives a low error rate when applied to this held-out dataset.

How do we get this test set?

  • First thought: divide the data randomly into a training set and a test set, where we build the classifier on the training set and test it on the test set.

Good thought!

  • However, the division of the data into the two sets is random, and we would get different results based on different divisions of the data.
  • So, we can repeat the splitting of the data into training and testing sets many times and average the error rates we get.
    • This is the idea of cross-validation.

Estimating Tuning Parameters¶

Many classification methods contain parameters that we need to tune:

  • kNN: choose the number of nearest neighbors to use
  • glmnet: choose the value of $\lambda$
  • PCR: choose the number of principal components to use

We want to choose the values of these tuning parameters that gives the lowest error rate.

We have the same problem as above:

  • If we use the same data to fit models with various values of the tuning parameter and to test their accuracy, we will underestimate the error of the model.
  • So: we want to choose the values of these tuning parameters that gives the lowest error rate in some held-out testing set.
    • We will also use cross-validation to get the best tuning parameter values.

The CV Process¶

Let's consider two cases:

  1. The classification method has no tuning parameters -- we only need to estimate the error rate.
  2. The classification method does have a tuning parameter -- we need to find the best tuning parameter value and estimate the error rate.

No Tuning Parameters¶

This is the basic framework of cross-validation.

Classifiers that might use this type of cross-validation are logistic regression and LDA.

for i in 1:10 pull out a test set fit the model to the training set apply the model to the test set and calculate the error compute the average of the 10 errors to get the error estimate

In Words:¶

  • Split the data into tenths.
  • Select nine of these tenths to use at the training set and one tenth as the testing set.
  • Use the training set to build the classifier.
  • Apply the classifier to the held out testing set and calculate the error rate.
  • Repeat the above steps, each time holding out a different tenth of the data as the testing set.
  • In the end, average the 10 error rates to get the final estimate of the error rate.

Things to Consider¶

Dividing the Data¶

The first step in the loop is to pull out a test set.

  • The conventional method is to split the data into 10 (basically) equal parts and hold one out each time.
  • It is also okay to take a different random subset each time.

k-fold CV¶

The number of times we split the data into a training set and a testing set if called the number of folds.

  • The previous example showed 10-fold CV.
  • Typically, cross-validation is done with $k = 5$ or $k = 10$.
  • There is a special case called leave-one-out CV, where $k = n$.
    • In leave-one-out CV, each test set contains a single observation and the classifier is fit on the remaining $n-1$ observations in the training set.
    • Leave-one-out CV can be computationally intensive, especialy when $n$ is large, but the error estimate will always be the same -- there is no randomness.

The Final Classifier¶

  • The final model will be fit using ALL the labeled data we have.
  • In this setting, the process of cross-validation is just a process for getting a good estimate of the error rate.

Tuning Parameters¶

This method uses k-fold CV in a nested manner -- need to do CV to pick the tuning parameter ($\lambda$) AND to estimate the error rate.

Classifiers that might use this type of cross-valiation are kNN, glmnet, and PCR.

for i in 1:10 pull out a test set for m in some set of lambda values for j in 1:10 pull out a validation set fit the model with tuning parameter m apply the fitted model to the validation set and calculate the error compute the average of the 10 validation errors for lambda = m pick the lambda that gives the smallest validation error use that lambda to fit the model to all the data, excluding the test set apply the fitted model to the test set and calculate the error compute the average of the 10 errors to get the error estimate

In Words:¶

  1. Specify a vector of parameter values to test. Call this vector $M$.
  2. Split the data into tenths.
  3. Select nine of these tenths to use at the training set and one tenth as the testing set.
  4. Select a value $m$ from the tuning parameter vector.
  5. Split the training set into tenths and pull out one tenth as the validation set.
  6. Use the smaller training set (without the validation set) to build the classifier with the parameter equal to $m$.
  7. Apply the classifier to the held out validation set and calculate the error rate.
  8. Repeat steps 5 - 7, each time holding out a different tenth of the original training data as the validation set.
  1. Average the 10 error rates to get an estimate of the error rate for the tuning parameter equal to $m$.
  2. Repeat steps 4 - 9 for all values in $M$.
  3. Choose the value of the tuning parameter that gives the smallest error rate.
  4. Use that value of the tuning parameter to fit the model to the entire original training set.
  5. Apply this classifier to the testing set and calculate the error rate.
  6. Repeat steps 3 - 13, each time holding out a different tenth of the data as the testing set.
  7. In the end, average the 10 error rates to get a final estimate of the error rate.

Choosing the Tuning Parameter¶

The CV done above allows us to get an accurate estimate of the error rate of our classifier. But how do we choose the best value of the tuning parameter $\lambda$?

  • Use the $\lambda$ values from the nested for loops above.
    • Since we find the best $\lambda$ value for 10 splits of the data, this process could potentially give us 10 different $\lambda$ values.
    • However, these values should be similar and we can just choose one (mean, median, mode) as our final $\lambda$.
  • Do CV again.
    • Just do the inside for loop in the above procedure to choose the $\lambda$ with the smallest error.

Choosing $k$ in kNN¶

kNN Refresher¶

  • For an observation $\tilde{x}$ in the testing set, find the $k$ points in the training set that are closest to $\tilde{x}$.
  • Classify $\tilde{x}$ based on the majority rule of the chosen $k$ nearest neighbors.

How do we choose the number of neighbors ($k$) we should consider?

M <- 1:20    # create the vector of tuning parameters
n <- nrow(X)

# create a vector to hold the 10 error rates
cv.error <- vector(mode = "numeric", length = 20)

# create a vector to hold the 10 best values of k
best.k <- vector(mode = "numeric", length = 20)

for(i in 1:20){
    
    train.ix <- sample(1:n, (19/20)*n)
    Xtrain = X[train.ix,]
    Ytrain = Y[train.ix]
    Xtest = X[-train.ix,]
    Ytest = Y[-train.ix]
    
    m.error <- vector(mode = "numeric")
    
    for(m in M){
        
        inside.error <- vector(mode = "numeric", length = 20)
        
        for(j in 1:20){
            n2 <- nrow(Xtrain)
            train2.ix <- sample(1:n2, (19/20)*n2)
            Xtrain2 = Xtrain[train2.ix,]
            Ytrain2 = Ytrain[train2.ix]
            Xvalidate = Xtrain[-train2.ix,]
            Yvalidate = Ytrain[-train2.ix]
            
            fit <- knn(train = Xtrain2, test = Xvalidate, cl = Ytrain2, k = m)
            inside.error[j] <- mean(fit != Yvalidate)
        }
        
        m.error <- append(m.error, mean(inside.error))
    }
    
    # find the k with the smallest error
    idx <- which.min(m.error)
    best.k[i] <- M[idx]
    
    fit.train <- knn(train = Xtrain, test = Xtest, cl = Ytrain, k = M[idx])
    cv.error[i] <- mean(fit.train != Ytest)
    
}

Choosing $\lambda$ in glmnet¶

Regularized Logistic Regression Refresher¶

We will introduce a penalty into the parameter estimation process to make the estimation better. We will look at the lasso method, whose penalty is related to the magnitude of the coefficient estimates.

  • Fitting a logistic regression model is a minimization problem -- we want to find the coefficient estimates that minimize the loss function.
  • In lasso, we introduce a penalty term that increases the objective function when the magnitude of the coefficients are large.
  • Therefore, to minimize the objective function, the coefficients need to help us predice the response (loss function part) and they need to be small (penalty part).
    • This causes the coefficients of some/many predictors to become 0.
  • There is a tuning parameter $\lambda$ that controls how important it is to keep the coefficients small.
    • We choose $\lambda$ through CV.
